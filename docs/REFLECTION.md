This project asked me to work with non-tabular and somewhat unstructured data, so I decided to create a scraping project centered around the BITMAN program pages and course pages. I mainly used AI to help with setting up my repository, organizing documents and folders, troubleshooting code, and fixing errors along the way. It also helped me map out the steps I needed to follow and figure out what would make my repo and project stronger overall.
One thing I liked about using AI was that it made the project feel more manageable and less stressful. Instead of looking at the work as one huge overwhelming task, it helped me break everything down into smaller, more digestible pieces. It was useful for setting up the right folder structure, getting my terminal ready, helping with coding, and working through errors when I got stuck. It was especially helpful during moments when I hit obstacles and did not know how to move forward.
At the same time, using AI was not always smooth. Sometimes the instructions were unclear or confusing, and other times the code it gave me would mess up the formatting, which meant I had to go back and fix parts of it myself. I tried to do most of the coding on my own unless I was genuinely stuck. In those cases, I used ChatGPT more as a guide to point me in the right direction and give me suggestions for how to make the project work better.
Overall, this project taught me that AI can be very useful in some situations, but not as helpful in others. It is not a replacement for thinking or problem-solving, but it can be a strong support tool when used the right way. I also learned a lot more about GitHub, file structure, coding, and the challenges of working with messy website data. If I were to do a similar project again, I would want to spend more time breaking down the code and expanding the project by adding more classes and courses.
